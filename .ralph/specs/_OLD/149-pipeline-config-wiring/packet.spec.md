# Packet Contract: 149-pipeline-config-wiring

## Goal

Close four pipeline-wide red tests in `crates/slicer-runtime/tests/arachne_parity.rs` (rewritten where applicable to drive behavior natively) by registering the missing OrcaSlicer config keys — `detect_overhang_wall`, `overhang_reverse`, `overhang_reverse_internal_only`, `extra_perimeters_on_overhangs` (re-publish), `min_width_top_surface`, `alternate_extra_wall`, `bridge_flow` (new), `thick_bridges` (new) — implementing D3 (`alternate_extra_wall` bumps `ArachneParams.max_bead_count` on odd layers, mirroring OrcaSlicer's `loop_number++ → inset_count → max_bead_count` beading-stack mechanism) and D4 (`bridging_flow(bridge_flow_ratio, thick_bridges)` returns the ratio for bridge vertices, mirroring OrcaSlicer's real formula NOT a fabricated 0.85 constant). D-104f (concentric infill Arachne wiring) is registered as an open deviation; the corresponding red test stays red. D-104g (per-vertex `flow_factor` divergence from OrcaSlicer's per-path `Flow` model) is added as an open deviation per the grilling decision.

## Scope Boundaries

This packet lands the four pipeline-config gaps (G8, G9, G14, G16 in the audit) plus the D3/D4 behavior changes, and registers D-104f and D-104g as open deviations. It edits the arachne manifest (11 new `[config.schema.*]` sections: the 8 audit keys plus `spiral_vase` + `sparse_infill_density` for the D3 gate and `only_one_wall_top` for the AC-2 read — none of the three is registered in the arachne manifest today), the classic manifest (7 new sections: 4 missing keys + `alternate_extra_wall` + `bridge_flow` + `thick_bridges`), the perimeter modules (D3 alternating logic + D4 bridge flow + the AC-2 source reads), `crates/slicer-core/src/flow.rs` (new `bridging_flow` helper), the AC-4 red test in `crates/slicer-runtime/tests/arachne_parity.rs` (rewritten — its host-junction predicate cannot observe the guest-side fix), and `docs/DEVIATION_LOG.md` (6 new rows: D-104b/c/d/e/f/g). It does NOT touch the concentric infill path (D5 / D-104f is deferred per the user's decision; the corresponding red test stays red). It does NOT touch the host pipeline's hardcoded `flow_factor = 1.0` / `overhang_quartile: None` (the guest module overrides per-vertex from `region.bridge_areas()` / `region.overhang_quartile_polygons()`, per packet 148).
