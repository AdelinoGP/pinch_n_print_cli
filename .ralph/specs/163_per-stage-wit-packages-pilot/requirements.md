## Problem Statement

A WIT world is an all-or-nothing contract. wasmtime's generated `Indices::new` resolves and typechecks **every** export in the world at `instantiate` time, so a guest must satisfy the world's entire export surface. `#[slicer_module]` pads the gap: `build_postpass_world_glue` emits `impl Guest for __SlicerPostpassComponent` with **both** `run_gcode_postprocess` and `run_text_postprocess`, and its own source comment concedes that whichever stage the module did not implement "returns a benign `Ok` so the component remains WIT-conformant". Those stubs are load-bearing padding that silently return success — a direct violation of ADR-0015's "do not swallow" — and they make any change to any export in a tier invalidate every guest in that tier.

ADR-0045 (accepted) fixes this by giving each stage its own versioned WIT package, so wasmtime's `alternate_lookup_key` major-track matching (verified against pinned `wasmtime-environ-43.0.1/src/component/names.rs`) does the compatibility work for free, and a module that declares a stage whose interface it does not export fails at load rather than no-oping. ADR-0044 already proved the current `wit-world` string enforces nothing; ADR-0045 is what gives the version teeth.

**The mechanism itself is no longer in doubt — its contact with our real WIT surface is, and that is what this packet exists to settle.** ADR-0045 §"Verified empirically, not just read" records a throwaway spike that executed the mechanism against the pinned wasmtime 43.0.1 / wit-bindgen 0.57.1: all five semver rows behave as claimed (`guest@1.0.0`/`host@1.5.0` resolves via the `@1` alt-key, in both directions; `@2.0.0` and `0.x` fail cleanly at `instantiate`); stage isolation reproduces literally (the unrelated guest's `sha256` is byte-identical across a sibling's major bump, never rebuilt); and the rejected "interfaces in a tier package" alternative fails on a control, replaying packet 130 exactly. **But the spike used no host imports at all.** Our stage worlds import `slicer:common/host-services`, `slicer:config/config-types` and (transitively) `slicer:ir-handles/ir-handles`, and they depend on `with:`-mapped resource **identity** holding across separate `bindgen!` calls — ADR-0002 remaps every non-layer world onto the layer world's one Rust type set; ADR-0003 explains why guests cannot. Splitting 4 `bindgen!` calls into 17, each `with:`-mapping the same dep interfaces, is the untested risk; if identity breaks, it breaks as the linking failure `CLAUDE.md` §"WIT/Type Changes Checklist" describes (`list<object-id>` in one file vs `list<MeshObjectView>` in another).

**163 therefore exists to prove the mechanism survives contact with real imports and `with:`-mapped resources on the cheapest tier, before 164 commits the other 13 stages.** It builds every moving part — canonical per-stage `.wit` packages, `slicer_schema::STAGES` package columns, per-stage host `bindgen!`, per-stage macro glue, fatal-on-miss dispatch, per-stage guest staleness in `xtask` — and exercises them on the three smallest stages (2 postpass + 1 finalization, 3 exports total). AC-12 is the falsifier: real typed instantiations plus real host↔guest resource round-trips through each pilot stage, never a decode-only check, because a decode check passes even when resource identity is broken. If AC-12 cannot be made green, the finding invalidates packet #3's premise and must be returned rather than worked around.

**Authoring AC-12 surfaced the sharpest instance of the thing this whole queue is about, one layer up.** The first draft of AC-12 listed `postpass_gcode_boundary_tdd.rs` among its round-trip proofs. That test's helper takes `_component: Arc<WasmComponent>` (`:63`) and discards it; it calls the dispatcher with `WasmInstancePool::placeholder()` and `None` (`:190-191`), which returns `Err(DispatchError { phase: DispatchPhase::MissingComponent })` before any linker, store, resource push or `instantiate` (`dispatch.rs:1014`); that error is converted to `Ok(PostpassOutput::GCodeSuccess)` by four un-`cfg(test)`-gated sites (`dispatch.rs:1721`, `:1925`, `:1970`, `:2017`); and the test then asserts the commands equal the exact vec it fed in. No WASM executes. It passes with the guest deleted, and it would pass with `with:`-mapped resource identity broken — the precise failure AC-12 exists to detect. A test named for carrying payload variants *into the guest* asserts the opposite of what it does.

**That leading underscore was a compiler unused-variable warning that someone silenced instead of heeded. The warning was the bug report.** This packet exists to delete `Ok(())` stubs that lie about doing work; resting its headline AC on a test that lies about doing work would have been the identical defect, one layer up — and it would have certified the per-stage mechanism on evidence that never ran it. AC-12 therefore rests on four suites each verified to assert on a value only a live guest can produce (a 1→8 command expansion, a guest-only `"[stamped] "` prefix, an observable 1.0 → 0.5 host-IR mutation). `postpass_gcode_boundary` is excluded by name, with the reason recorded in `design.md` §"Why `postpass_gcode_boundary` is not in AC-12" so nobody helpfully re-adds it.

Stated precisely, because the tempting overstatement is wrong: the component skip is **deliberate, not a bug** — `binding.rs:29` documents `CompiledModuleLive::wasm_component` as "`None` for placeholder or test modules", and the four conversion sites are load-bearing for host-builtin stages that legitimately have no component. The **proven** defect is only the vacuous test; whether a real module whose component failed to load can also reach `None` and be silently skipped (an ADR-0015 "do not swallow" violation) is **untraced**, and this packet asserts nothing about it. The defect is already filed and committed as **`DEV-087`** (`57ceae39`), which carries this chain and marks the production question as untraced. This packet **references** `DEV-087` and files no row for it; it only stops relying on the test.

Grounding this packet surfaced one shape that neither ADR-0045's prose nor its spike covers, and that a naive reading would get wrong. The spike's `interface foo { hello: func() -> string; }` takes no parameters; our stages take **host-owned resources**. `world-postpass.wit` and `world-finalization.wit` declare `gcode-output-builder`, `layer-collection-view` and `finalization-output-builder` in the **world body**, which makes them host-implemented — `host.rs::postpass_impls` really does `impl ppm::HostGcodeOutputBuilder for HostExecutionContext`. A resource declared in an **exported** interface is owned by the **guest**. Folding the world body straight into the exported interface would therefore invert ownership and break every host builder impl. Each resource-bearing stage package consequently pairs an **imported** `<iface>-types` interface (holding the resources and plain data types) with the **exported** interface (holding only `run`) — the same shape `world-layer.wit` already uses by taking its builders from the imported `slicer:ir-handles/ir-handles`. Both interfaces sit in one package, so there is still exactly one version per stage.

A second premise of the approved plan was **falsified while grounding this packet** and is corrected here: the plan and its packet-queue row assume the ADR's headline outcome ("untouched, doesn't even rebuild") follows from packaging alone. It does not. `xtask/src/build_guests.rs::compute_shared_mtime` folds every `.wit` under `crates/slicer-schema/wit/` into a single mtime that `is_stale` applies to **every** guest, so touching one stage's WIT would still mark all 32 guests `STALE` no matter how the packages are cut. Delivering the ADR's claim requires splitting that WIT input per stage; this packet therefore owns the `xtask` change, without which its own headline negative case (AC-N2) is unprovable and packet #3 would inherit the same hole.

No packet is reopened or superseded. `162_wit-lifecycle-export-removal` is a hard prerequisite, not a predecessor to absorb.

## In Scope

- Replace `crates/slicer-schema/wit/deps/world-postpass/` and `crates/slicer-schema/wit/deps/world-finalization/` with three per-stage package directories under `crates/slicer-schema/wit/deps/`: `postpass-gcode-postprocess/`, `postpass-text-postprocess/`, `finalization-layer-finalization/`. Each holds one `package slicer:<dir>@1.0.0;`, one `run: func`, one `world <dir>-module`. The two resource-bearing packages each hold an **imported** `<iface>-types` interface (carrying `gcode-move-cmd`, `retract-mode`, `gcode-retract-cmd`, `gcode-fan-speed-cmd`, `gcode-temperature-cmd`, `gcode-tool-change-cmd`, `resource gcode-output-builder`, `gcode-command`; and `layer-idx`, `object-id`, `region-id`, `region-key`, `tool-change-view`, `print-entity-view`, `z-hop-view`, `entity-mutation`, `sort-key`, `synthetic-layer-data`, `resource layer-collection-view`, `resource finalization-output-builder`) plus an **exported** interface holding only `run`; `postpass-text-postprocess` takes no resource and is a single exported interface. ## Step Completion Expectations

- **Ordering is load-bearing.** The WIT packages (Step 1) must land before the schema columns (Step 2), which must land before host bindgen (Step 3), macro (Step 5) and `xtask` (Step 7) — all three read `STAGES`. The tree does **not** compile between Step 1 and Step 6; do not treat an intermediate `cargo check` failure as a defect. The first green `cargo check --workspace --all-targets` is Step 6's exit condition, and that is the earliest point any test result means anything.
- **Every WIT / macro / schema edit invalidates all 32 guests.** After Steps 1, 2, 5 and 6 the implementer must run `cargo xtask build-guests` (not `--check`) before running any test that instantiates a component. Step 7 changes the staleness rule itself; re-run `cargo xtask build-guests` once more after it so the recorded artifact mtimes are consistent with the new rule.
- **AC-N2 depends on Step 7 and Step 1 jointly** and cannot be evaluated before both land. It is destructive to guest mtimes (it `touch`es a `.wit`); run `cargo xtask build-guests` afterwards to restore a clean `--check`.
- Shared scratch: `target/test-output.log` is overwritten by every run per `CLAUDE.md`. Capture findings before launching the next run. AC-N2 writes `target/stale.txt`.
