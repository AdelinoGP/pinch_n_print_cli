# Packet Contract: 177-arachne-baselines-to-structural-invariants

## Goal

Convert the Arachne self-captured baseline corpus into ADR-0042 structural invariants anchored by a measured coverage-ratio-vs-classic gate, so that a D5-class drift fails the suite instead of passing it.

## Scope Boundaries

Untangles the odd `max_bead_count` defaults that entangle the tapered-wedge baselines, then converts the `tapered_wedge` perimeter-parity fixture (**green today, and already re-blessed to the post-D5 value by `9ca62ba0` — see the `[BLOCK]` in `design.md`; the packet was briefed that it is RED and it is not**) and the eight `crates/slicer-core/tests/fixtures/arachne/*.json` baselines into structural assertions, extending the existing `crates/slicer-core/tests/arachne_invariants.rs` home. Surviving self-captured data is demoted to explicitly-labelled change-detectors; the red-test files are rehomed and the stale `arachne_parity.rs` header corrected. No production Arachne geometry behaviour is changed beyond the `max_bead_count` default correction, and no fixture is re-captured to make a red test green.

## Prerequisites and Blockers

- Depends on: ADR-0042 (`docs/adr/0042-arachne-parity-structural-invariants-over-fixtures.md`, accepted); the landed `max_bead_count` module-manifest sentinel fix recorded in `docs/specs/arachne-parity-recovery.md`.
- Unblocks: closure of `D-112-SELFCAPTURED-BASELINES` in `docs/DEVIATION_LOG.md`; retirement of the "no OrcaSlicer oracle" blanket acceptance for the Arachne unit corpus.
- Activation blockers: **two `[BLOCK]`s in `design.md` §Open Questions, both gating activation.** (1) The change-detector CI-authority decision — fail-on-drift vs warn-only — which **Step 7's precondition depends on**; the packet cannot execute past Step 6 without it. (2) The measured-threshold conflict: if Step 2's `observed_min - margin` would admit `0.668`, AC-3 and AC-4 are in direct conflict and the corpus itself is implicated — escalate, never tune. Both require a maintainer decision; status stays `draft` until (1) is answered.

## Acceptance Criteria

State ACs only here; `requirements.md` references their IDs.

**Every `cargo test` command below MUST pass `--features host-algos`.** Not polish, not style. `crates/slicer-core/tests/arachne_invariants.rs` carries `#![cfg(feature = "host-algos")]`; `host-algos` is **not** in `slicer-core`'s `default` features; and — unlike the `algo_*` targets — `arachne_invariants` has **no `required-features` entry in `crates/slicer-core/Cargo.toml`** to catch the omission. Verified at authoring time: without the feature, `arachne_invariants`, `propagation`, `centrality`, `bead_count` and `generate_toolpaths` each collect **zero tests, print `test result: ok. 0 passed`, and exit 0**. A green run proves nothing.

That is why every AC below pairs the feature flag with a **count gate** (`1 passed`, or a `0 passed; 0 failed` vacuity check) instead of trusting an exit code. An AC that cannot tell "passed" from "never ran" is precisely the self-ratifying instrument this packet exists to eliminate — the D5 failure mode wearing a CI badge.

- **AC-1. Given** the two production defaults `ArachneParams::default()` (`crates/slicer-core/src/arachne/pipeline.rs`) and `BeadingFactoryParams::default()` (`crates/slicer-core/src/beading/factory.rs`), **when** the `max_bead_count` correction lands, **then** each initializes `max_bead_count` to `10` — an even value, matching canonical's always-even `2 * inset_count` derivation in `WallToolPaths.cpp::generate` — and a test asserts `ArachneParams::default().max_bead_count % 2 == 0` and `BeadingFactoryParams::default().max_bead_count % 2 == 0`. Note: `inset_count` is **not** a field of either struct and does not exist anywhere in `crates/slicer-core/src/`; it is canonical's derivation input, reachable here only via config, so the assertion is on evenness and the literal, never on a computed `2 * inset_count`. | `cargo test -p slicer-core --features host-algos --test arachne_invariants -- production_defaults_max_bead_count_is_even --exact 2>&1 | rg -q 'test result: ok. 1 passed' && echo PASS || { echo FAIL; false; }`

The command must distinguish a filled table from the empty skeleton, so it greps for a digit after the threshold label, not for the heading. | `rg -q '\*\*Chosen threshold = observed_min - margin:\*\* [0-9]' .ralph/specs/177-arachne-baselines-to-structural-invariants/design.md && echo FILLED || { echo EMPTY-FAIL; false; }`
